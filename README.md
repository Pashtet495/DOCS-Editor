# DOCS-Editor
A SuperDOC-based text editor that packages DOCX-compatible documents into the new DOCS container format, which includes a document map with preserved embedding vectors and the ability to store ts and js scripts used in the document. Designed for collaborative editing (human + AI) and for RAG searching without creating external vector databases.

**При создании форков или иных копий учитывайте лицензию на использование SuperDOC редактора**.

# Версия 0.2 alpha
Данный релиз не является стабильной версией приложения. Текущая версия обеспечивает возможность использования планируемого функционала. Релиз направлен на ознакомление с функционалом и фитбеком для улучшения функционала. В текущей версии не предусмотрены многие исключения и блокировки действий пользователя (например создание пользовательских переменных с названиями адресов ячеек табличного реактора, создание циклических зависимостей в формулах и тому подобное), которые могут приводить к некорректной работе приложения.
### Ключевые изменения (встроил в текстовый редактор подобие Excel и mathCAD с синхронизацией формульного, табличного и текстового содержимого).

- Текстовый редактор разделён на два режима: режим редактирования и режим просмотра (переключатель в верхней части интерфейса).

-  В режиме редактирования введена возможность вставлять интерактивные canvas подобные элементы, которые в режиме чтения рендарятся в изображения. Если раскрыть docs сохранение через Zip архиватор, то там будет файл "view.docx" - это полностью совместимый с MS Word файл в котором все динамические элементы заменены на текст и статические изображения.
- В правой панели основного интерфейса во вкладке ресурсы введено 3 элемента "Таблицы", "Калькулятор", "Автозаполнение".
<img width="378" height="524" alt="002" src="https://github.com/user-attachments/assets/0c391deb-e387-410c-8efa-0897dc9ee02d" />

- Теперь DOCS позволяет в единый документ записывать и текстовые документы и таблицы. Добавлен встроенный табличный редактор с базовым функционалом. Внутри документа можно хранить неограниченное количество таблиц.
<img width="2550" height="1378" alt="003" src="https://github.com/user-attachments/assets/78570c7e-ed37-48aa-b37f-719eb62cbecd" />

- Текущие функции табличного редактора: хранение текстовых/цифровых данных в ячейках, поддержка базовых математических операций с ячейками в стиле MS Excel (ячейки с содержимым начинающимся на "=" считаются формулами). Реализована топологическая сортировка выполнения операций (табличный редактор выстаивает правильную последовательность выполнения операций). Растягивание ячеек для создания копий с поддержкой изменений относительных и абсолютных координат в формулах (используется стандартный функционал литеры $ при использовании с координатами ячеек как в MS Excel). Возможно добавление строк/столбцов с автоматическим изменением координат в формулах, так чтобы ссылки в формулах не ломались (как в Excel).
- Текущие функции режима "калькулятор". Проведение математических расчётов. Логирование истории операций. Возможность использовать значения и формулы калькулятора в качестве глобальных формул и переменных. Пересчёт всего пайплайна расчётов при корректировке сохранённых переменных или формул расчёта (как в MathCAD). Возможность вставлять в документ LaTeX формулы из калькулятора. Возможность настраивать нумерацию формул в документе. Возможность вывода в формате "имя функции = формула = значение", "формулы = значение", "Имя функции = значение". Возможность определять назначение переменных и функций в графе комментарий. Возможность автоматической генерации раздела "где: ...." с описанием переменных используемых в функции при вставке формулы в документ. Функции автоматически перестраиваются по всему документу, если корректировать их через калькулятор. При переключении редактора в режим просмотра LaTeX функции автоматически рендерятся в изображения (docx документ из файла сохранений можно открывать в MS word, но там формулы будут в виде изображений). Встроен автоматический парсер LaTeX содержимого из документа. Формулы записанные текстом в документ автоматически заменяются на LaTeX, автоматически добавляются в калькулятор, нумеруются. В калькуляторе есть возможность настройки по умолчанию отображения внешнего вида блока формул распознаваемых парсером. Итоговая задача функционала: создание расчётных/научных/методических/учебных документов с красивыми формулами. Автоматический пересчёт всех формул (как в MathCAD) и их отображения в документе при изменении переменных или формул.
- Синхронизация режима табличного редактора и калькулятора. В табличном редакторе есть кнопки "f" и "f+". Кнопка "f" позволяет синхронизировать значения любой ячейки таблицы с формулами или константами заданными через калькулятор. Табличный редактор сам определяет задана в калькуляторе формула или константа. Для констант допускается задание данных для константы из таблицы. В калькулятор добавляется значение ячейки. Для формул табличный редактор определяет все зависимые переменные и предлагает их синхронизировать с ячейками таблицы. Таким образом можно выполнять какие-то расчёты в табличном редакторе а потом отдельные ячейки задавать как глобальные переменные. Кнопка f+ позволяет создавать новые формулы и константы не выходя из режима табличного редактора. При использовании табличного редактора для создания формул дополнительно доступно создание переменных в виде векторов и матриц, привязанных к ячейкам таблицы. Это позволяет в калькуляторе реализовывать более сложные операции линейной алгебры.
- Режим автозаполнения позволяет создавать в документе метки, которые в режиме просмотра документа автоматически заменяются на настроенное содержимое.

Добавление меток и их значений возможно как в правой панели редактора, так и через специальный табличный редактор.

<img width="1842" height="939" alt="004tab" src="https://github.com/user-attachments/assets/2c53671d-0988-407d-9a83-2dc67afad1c1" />

Можно задавать ограничивающие списки между связными полями (например список сотрудников и список табельных номеров сотрудников, тогда при выборе номера для одного поля автозамены в другом поле автоматически будет предложен нужный ФИО или наоборот при выборе ФИО будет предложен нужный номер). Возможно создание групп вставок с идентичными наборами заполнений (как в примере выше - первое поле содержит варианты сортаментов, второе поле варианты размеров проката для всех вариантов сортаментов, третье поле содержит отрезные размеры для всех вариантов сортаментов. Через связи создаются ограничения, допустим если в редакторе активны листы или плиты, то в поле для вставки габаритов доступны только габариты для плит и листов).

<img width="335" height="401" alt="004svaz" src="https://github.com/user-attachments/assets/f9561b0b-ab10-402b-abc4-cbc21ac74747" />
Значения в полях можно синхронизировать с ячейками в таблицах. Например сделать таблицу в которой проведены расчёты для сметы и ячейка с итоговой суммой задаётся в качестве ссылки для автозаполнения текстового документа. (Для этого в табличном редакторе есть кнопка линии с цепочкой).
<img width="255" height="91" alt="005" src="https://github.com/user-attachments/assets/7e0efc1c-70ac-47b1-8f05-9b72ce77774e" />
В таком случае можно сделать в документ выставления счёта автовставку, синхронизированную с ячейкой таблицы. При изменении значения в ячейке таблицы - текстовый документ счёта корректируется автоматически. В общем - автовставка позволяет создавать мосты между текстовым и табличным документами (как будто MS Word и MS Excel были бы одной общей программой, оперирующей общими данными).
Можно сделать кроссинхронизацию одной ячейки таблицы и с формулами и с автозаполнением.

### Технические изменения

Отшито использование next.js в релизной версии. Вся работа реализована через Electron API. В dev моде для отладки сохранено использование работы приложения через next.js сервер.

### Состояние реализации ИИ помощника

В режим калькулятора ИИ ассистент добавлен, но работает очень плохо. В режиме таблиц ИИ функции пока не реализованы (предполагается разбивка таблицы на зоны, кнопка для этого в интерфейсе уже есть. Зоны будут отдельно прописываться в карту документа для эффективной работы с нейросетями). В режиме текстового редактора стабильность ИИ помощника ухудшилась в сравнении с прошлой версии из-за добавления функционала к которому у нейросети доступа пока нет.
